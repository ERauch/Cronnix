ライセンス

当プログラムは完全に無料です。また、当プログラムにより発生したいかなる事象に関しても、作者は一切責任を負いません。

連絡先

ご提案、ご不満、何であれ、当プロジェクトに関するフィードバックは常に歓迎します。作者のアドレスはこちら： sas@abstracture.de

CronniX の配布について

CD-ROM などの配布メディアに CronniX を収録したい場合には、ご遠慮なく私にご一報ください。CronniX の再配布は自由ですが、どこでどのように配布されるのかに興味があるのです。

CronniX の記事を書いたり文中で触れる場合にも、何か質問があれば（なくても知らせる意味で）メールをください。

2.0 - 2003/01/18
  • 改良された、ダイアログベースのタスク作成
  • タスクのコメント
  • Column reordering (with saved state)
  • オンラインでのアップデートの確認（手動）
  • バグ修正：crontab が破損しているときの起動時のクラッシュ
  • ヘルプを更新
  • MDI への準備として内部のデータモデルを再構築
1.5.1 - 2003/09/27
  • Jaguar でのヘルプのバグを修正：ヘルプを表示できるようにした
1.5 - 2001/11/11
  • タスクを即座に実行できる“今すぐ実行”コマンド
  • タスクの使用／停止を切り替えるチェックボックス
  • 削除のショートカット
  • タスクのコンテクストメニュー
  • 削除／複製の複数選択対応
  • フランス語ローカリゼーション（訳：Sylvain Cuaz）
  • 日本語ローカリゼーション（訳：E-WA）
1.4.1 - 2001/09/09
  • ついに！アイコンを新調。Forrest Walter に感謝。
  • バージョン文字列を修正。
  • ドキュメント中のミスタイプを修正。
  • ドイツ語ローカリゼーションを追加。CronniX をローカライズしたい方は作者までご連絡ください。
  • システムの crontab に“/etc/crontab”に加え“/>”と書き込んでしまうバグを修正。Unix な人なら何が起こるかわかるでしょう - 私はコメントせずにおきます ;-)
1.4 - 2001/08/02
  • 名前にスペースを含むプログラムを挿入する際、引用符で囲むようにした。
  • システム crontab のサポートを追加。
  • ほかのユーザの存在しない crontab を開こうとした際の問題を修正。
  • ツールバーを追加。
  • メニュー項目を追加。
  • ドキュメントを更新。
1.3 - 2001/06/03
  • ほかのユーザの crontab を開こうとして、crontab が存在しなかった場合、CronniX が「文句を言い過ぎる」バグを修正。
  • “ユーザの crontab を開く”を改善、“root”以外のユーザも（認証後に）ほかのユーザの crontab を開くことができるようにした。
  • 環境変数のサポートを追加。
1.2 - 2001/03/27
  • CronniX の再起動が速すぎて crontab の再読み込みが阻害されてしまうバグを修正。
  • ドラッグ＆ドロップ機能の補助としてメニュー項目“プログラムを挿入”を追加。選択したプログラム／スクリプト（実際にはあらゆるファイル）をコマンドとして挿入できるようになった。
  • インターフェースの細かな修正。ボタンのラベルを“Write”から“Save”に変更、ショートカット付きのメニュー項目“Save”を追加。
  • OS X 10.0（または最終版、商品版、GM、4k78…）上での動作を確認。
1.1 - 2000/12/26
  • ユーザからの提案により、ドラッグ＆ドロップに対応した。選択したテキストを行にドロップすると、コマンド入力欄にコマンドとして挿入される。ファイルをドロップすると、ファイル名の前に“open”をつけた形で挿入される。これは AppleScript を挿入するのに便利だろう。ドロップした内容が 1 行に収まらない表組のような場合は、新たな行が挿入される。
  • “ファイル”メニューの項目“ユーザの crontab を開く…”で開くダイアログから、ほかのユーザの crontab を開けるようになった。ただしこれにはスーパーユーザ（“root”）権限が必須。crontab はほかのユーザの crontab へのアクセスを拒否するが、スーパーユーザだけは例外。
  • オンラインヘルプ。
  • Objective-C で書き直した。これにより Java バージョンに比べ CronniX の起動が速くなり、また使用メモリ容量が減った。
1.0 - 2000/12/07
  • 最初のリリース（Java バージョン）
